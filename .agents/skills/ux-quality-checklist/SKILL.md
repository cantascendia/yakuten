# UX 质量检查清单

在提交任何涉及 UI 的代码变更前，逐项检查以下内容。未通过的项必须修复后再提交。

## 文本与排版
- [ ] 所有用户可见文本设置了溢出处理（ellipsis / fade / 换行）
- [ ] 动态文本（用户输入、服务端返回）测试过超长内容
- [ ] 字体大小、颜色、行高引用主题 / 设计 token，非硬编码数值
- [ ] 多语言场景下文本容器留有余量

## 交互与触控
- [ ] 所有可点击元素有视觉反馈（涟漪、高亮、透明度变化）
- [ ] 移动端可点击区域 ≥ 48x48 dp/pt
- [ ] 按钮在加载/禁用状态下不可重复点击
- [ ] 表单有输入验证且错误提示清晰

## 状态完备性
- [ ] 空状态：无数据时显示引导提示而非空白
- [ ] 加载态：异步操作有 loading 指示器
- [ ] 错误态：网络失败/接口异常有友好提示和重试入口
- [ ] 部分加载：骨架屏或渐进显示

## 布局与响应式
- [ ] 在最小支持屏幕宽度下无溢出（Flutter: 检查 overflow 黄条）
- [ ] 在最大屏幕宽度下内容不过度拉伸
- [ ] 横竖屏切换（如支持）无布局崩溃
- [ ] 安全区域（刘海、底部手势条）已处理

## 性能感知
- [ ] 长列表使用懒加载构建（ListView.builder / Sliver 等）
- [ ] 图片加载有占位符，使用缓存
- [ ] 页面切换有合理过渡动画
- [ ] 无明显卡顿（避免在 build 方法中做复杂计算）

## 数据真实性
- [ ] 无硬编码假数据残留（搜索 "TODO"、"test"、"lorem"、"假"、"mock" 等关键词）
- [ ] 所有显示数据来自状态管理 / API / 本地存储
- [ ] 所有表单提交连接到真实处理逻辑
