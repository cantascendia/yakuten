# UX 质量检查清单

在提交任何涉及 UI 的代码变更前，逐项检查以下内容。未通过的项必须修复后再提交。

## 文本与排版

- [ ] 截断风险文本设置了 `overflow: hidden` + `text-overflow: ellipsis`，或 `overflow-wrap: break-word`，避免溢出容器
- [ ] 动态内容（服务端返回、用户输入）已验证超长内容不会破坏布局
- [ ] 字体引用使用 CSS 变量（`var(--font-display)`、`var(--font-body)` 等），不写死 `font-family` 具体值
- [ ] 多语言环境下行高（`line-height`）适配各语言字符高度（中文/日文建议 ≥ 1.7，英文 ≥ 1.5）
- [ ] 标题层级语义正确（`h1` 唯一，`h2`/`h3` 按文档结构递进），不用标题标签做视觉装饰

## 交互与点击目标

- [ ] 所有可点击元素有 `:hover` 和 `:focus-visible` 视觉反馈（颜色变化、下划线、阴影等）
- [ ] 移动端可点击区域 ≥ 44×44px（可用 `min-height`/`padding` 扩大热区，不必改变视觉大小）
- [ ] 禁用状态元素有明确样式（`cursor: not-allowed`、降低不透明度），且不可触发交互
- [ ] 表单字段有输入验证，错误消息位于字段附近且语言随 locale 切换

## 状态完备性

- [ ] 空状态：列表/搜索结果为空时显示引导性提示文案，不展示空白区域
- [ ] 加载态：所有异步操作（API 请求、Pagefind 搜索）有明确的加载指示器（spinner 或骨架屏）
- [ ] 错误态：网络失败或逻辑异常有友好提示（避免裸露技术错误信息），并提供重试或跳转入口
- [ ] React Islands 使用骨架屏或渐进显示，hydration 前不出现空白跳动（CLS）

## 布局与响应式

- [ ] 在 320px 最小视口宽度下无横向滚动条，无内容溢出
- [ ] 正文内容有 `max-width` 约束（建议 ≤ 780px），在超宽屏幕下不过度拉伸
- [ ] Starlight 侧边栏在移动端可正常折叠/展开，不遮挡主内容
- [ ] `padding` / `margin` 使用 CSS 变量或相对单位，不写死像素值至边缘
- [ ] 设备安全区域已处理（`env(safe-area-inset-*)` 用于底部固定元素）

## 性能感知

- [ ] React Islands 使用 `client:visible` 懒加载（仅对需首屏立即交互的组件使用 `client:load`）
- [ ] 图片使用 WebP 格式，配置 `width`/`height` 属性防止布局偏移，超出首屏的图片加 `loading="lazy"`
- [ ] Pagefind 搜索索引在生产构建时已生成，搜索响应无明显延迟
- [ ] 所有动画仅使用 `transform` 和 `opacity` 属性，并提供 `@media (prefers-reduced-motion: reduce)` 回退（关闭或简化动画）

## 医疗内容 UX（yakuten 专项）

- [ ] 紧急情况 banner 使用红底白字，在页面中始终可见，不提供关闭按钮
- [ ] 证据等级徽章（A/B/C/X）颜色编码清晰可辨，色差足够通过 WCAG AA 对比度（≥ 4.5:1）
- [ ] `<CitationRef>` 链接可正常点击，指向真实 DOI 或可访问的参考文献
- [ ] 血检工具的交通灯颜色（绿/黄/红）配有文字标签，不依赖单一颜色传达风险信息（色盲友好）
- [ ] 药物剂量数据旁有来源标注，视觉上与正文有区分（如使用角标或引用样式）

## 数据真实性

- [ ] 无硬编码假数据残留（搜索关键词：`TODO`、`test`、`lorem`、`mock`、`假`、`占位`、`placeholder text`）
- [ ] 所有显示数据来源于 `src/data/*.json`、API 响应或真实内容文件，不存在内联的演示用常量
- [ ] 所有表单或交互按钮连接到真实处理逻辑（AI 问答、血检工具等），不存在 `console.log` 替代实现
