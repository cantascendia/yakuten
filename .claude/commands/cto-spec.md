# Spec-Driven 开发启动

启动一个新功能的 Spec-Driven 开发流程。

## 功能概述

$ARGUMENTS

## 执行步骤

1. 阅读现有产品愿景（docs/ai-cto/PRODUCT-VISION.md）和技术愿景，确认此功能与远景的关系
2. 起草 SPEC.md 内容（手册 §18 规范），包括：
   - 功能详细描述、目标用户、成功标准
   - 输入输出定义、API/数据模型设计、边界条件和异常处理
   - 非功能需求（性能、安全、兼容性要求）
   - 与现有模块的依赖关系
3. 创建 `docs/ai-cto/SPEC.md` 文件
4. 等用户确认 SPEC 后，生成 PLAN.md 和后续执行任务

如果 SPEC 撰写过程中发现产品愿景有遗漏或技术架构需要调整，直接提出。
