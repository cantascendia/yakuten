# Yakuten 文案风格 & 写作约束

> 本文档为 yakuten 项目的中文内容写作准则。任何人（包括 AI 助手）在向 `src/content/docs/zh/` 提交新页面或大规模改写前，应先过一遍此文档。

## 1. 网站定位

HRT药典 = **循证 · 减害 · 引导就医**。

**不是**：百科、论坛、购药渠道、个人化处方。
**是**：中文跨性别女性社区的「安全底线信息站」—— 在找到愿意接诊的医生之前，不让人死于信息缺失。

内容必须有证据背书：WPATH SOC 8 / Endocrine Society 2017 / UCSF 2016 / 同行评审文献。每条医学主张要能通过 DOI 追溯。

## 2. 目标人群

**中国跨性别女性 HRT 使用者**。共性：

- 绝大多数 DIY（无正式处方路径，自行获取日雌 / 印雌 / 补佳乐 / 色谱龙等）
- 年龄跨度极大（高中生 ↔ 中年）、知识水平两极（医学生 ↔ 无医学背景）
- 被大量「劝退型」「煽情型」「科普型」信息灌过，对居高临下的文案极度敏感
- 焦虑度高（身份 + 健康 + 亲友压力）
- 核心需求：**把我当成年人看，给具体数据，让我自己决定**

**由此推出的不可触碰底线**：

- ❌ 居高临下（"我们建议您..."）
- ❌ 煽情（"我们理解你的痛苦..."）
- ❌ 劝退（"你真的需要想清楚再..."）
- ❌ 遮掩风险（"虽然有一点风险但..."）
- ✅ 具体数字、剂量、天数、引用
- ✅ 假设读者**明天**就要做决定
- ✅ 尊重 DIY 现实（不假设读者有处方 / 医生跟踪）

## 3. Voice Fingerprint（8 条）

1. **证据驱动** —— 每个医学主张带 `<CitationRef>`。数据第一，修辞第二。
2. **无主语陈述为主 + 必要时第二人称** —— "超过以下剂量不会让你女性化更快"。不是 "你应当..."。
3. **章节标题 = 名词短语** —— "基准检查清单"、"绝对禁忌症"、"危险的药物组合"。
4. **保护性家长腔，但不说教** —— 比喻代替禁令（"等于蒙着眼睛过马路"）。错误归咎于「社区流传的方案」，不归咎于读者。
5. **警示靠组件分层** —— `DangerBox` / `WarningBox` / `ContraindicationBox` / `DoseTable`。**紧迫感由视觉承载，不靠词语堆砌**。
6. **中文全角标点** —— `"..."`、`——`、`·`。禁 ASCII `"`。
7. **专业词 + 品牌名直接用** —— 补佳乐、色谱龙、VTE、RR=1.48。**社区黑话（日雌 / 印雌）只在"买药"上下文使用**，正文术语用正式名。
8. **严厉度按页面定** —— `dose-limits` 最硬（"绝对禁止"）、`blood-tests` 最口语（"拿到报告了？"）、`risks` 在冷数据与警报间切换。

## 4. 写作规则

### A. 章节标题

- ✅ 名词短语："脑膜瘤风险"、"给药途径对比"、"基准检查清单"
- ❌ 问句："这条路径适合我吗"
- ❌ 祈使句 + 副标题："决定打针后 · 前置血检（打第一针前 1-2 周）"
- ❌ 装饰符（`→`、`·` 副标题、括号时间窗口）出现在标题里
- **不要用 `§` 符号** —— 这是西文学术 / 法律文书的 section sign，中文排版里不自然。`before-you-start.mdx` 用了是前作者个人习惯，不是全站惯例。其他药物页、dose-limits、risks 都**没用**。新页面一律用**纯中文名词标题**，不带 `§`、不带编号前缀、不带装饰 span。
- 不需要在 H3 里再套一层编号。章节内部引用用**章节名 + 自然语言**（例："见下方〈立即停药并就医〉一节"），不用 "§4.3" 这种写法

### B. 开头段

- ✅ 陈述句开头，必要时插一句第二人称设问
- ❌ 对话式："你要打第一针"
- ❌ 自我介绍 / 页面简介："本指南面向..."、"本页按时间顺序列出..."、"本页按起点分组..."
- ❌ 信息架构 meta："按时间顺序给出可执行步骤、任务流、决策点、核心内容："
- ❌ **作者视角抱怨 / 解释**："最常踩的坑是信息散在三四个页面"（读者不关心网站内容为什么散，读者只想解决问题）
- ❌ 解释 TOC / 章节结构（让标题自己说，不需要在正文里解释)
- **判别法**：如果一句话里出现"本页 / 本文 / 本指南 / 核心内容 / 按 X 排列 / 按 X 组织"，99% 是 meta，删掉重写
- 参考模板：`before-you-start.mdx` 开头 "不是劝退，而是安全检查。跳过这一步开始用药，等于蒙着眼睛过马路。" —— 一句带立场的陈述，直接切题。
- 另一参考：`dose-limits.mdx` 开头 "超过以下剂量不会让你'女性化更快'。" —— 反驳一个常见误解 + 给出正面立场，不解释自己。

### C. 用组件，不用纯 Markdown

| 场景 | 必须用 | 禁止 |
|------|--------|------|
| 禁忌列表 | `<ContraindicationBox type="absolute\|relative">` | `- ` bullet list |
| 基线血检项目 | `<BaselineChecklist>` | 纯表格 |
| 时间阶段 | `<PhaseBlock>` | `### Day 0` 子标题 |
| 决策分支 | `<DecisionNode>` / `<DecisionBranch>` | Markdown 流程图 |
| 剂量数字 | `<InlineValue value="100–200" unit="pg/mL" />` 或 `<DoseTable>` | `100-200 pg/mL` 裸文本 |
| 紧急危险 | `<DangerBox>` 红 | `> **警告**：` |
| 一般注意 | `<WarningBox>` 黄 | 同上 |
| 现实预期 | `<RealityCheck>` | bullet list |

**铁律**：列信息前先问「有没有既有组件能承载」。有就必须用。

### D. 标点

- **引号** `"..."` 全角弯引号（U+201C / U+201D）
  - 例外：嵌套时外 `""` 内 `「」`
  - 禁 ASCII `"`，禁单纯用 `「」` 作外层
- **破折号** `——` 两个全角连用（U+2014 × 2）
- **间隔号** `·`（U+00B7）只用于**名字分隔**（如 "循证 · 减害 · 引导就医"）、**Stitch 字段分隔**（如 DrugInfoBox 里 "肌肉注射 (IM) / 皮下注射 (SC)"）
  - 禁用作章节标题副标题引子（「§ 1 决定打针后 · 前置血检」违规）
- **箭头** `→` 禁用于文案装饰；表格单元格内、流程图、导航 microcopy（"查看指南 →"）内允许

### E. 表格

- **`<DoseTable>` 列数 ≤ 4**
- 剂量/数据列单元格：只放数字 + 单位（"2-4 mg/周"），描述性内容挪到正文或 WarningBox
- 行数 > 6 时考虑拆成两张表

### F. 语气强度分级

| 级别 | 用词 | 场景 |
|------|------|------|
| 医学禁忌 | 绝对禁止 / 禁止合用 / 不推荐 | 生命安全红线 |
| 技术建议 | 建议 / 首选 / 优先 / 应当 | 指南级推荐 |
| 个人决策 | 可以考虑 / 若... / 根据... | 有选择余地的场景 |

**不要把高级别用在低优先级内容上**。写反面例子："**完全不用抗雄**也是合法路径" —— "合法路径"过重，应为 "**只用 E2 注射、不加抗雄**，也是合理方案"。

### G. 禁用词表（PM 腔雷区）

**禁用**：

- 任务流、决策点、协议矩阵、切换协议、时序管理、方案意向、讨论要点
- 本指南面向 X 的使用者
- 按时间顺序给出可执行步骤
- 核心内容：X → Y → Z
- 以下内容包括
- 我们将从 A 到 B 分步讲解

**替代说法**：

| 禁用 | 替代 |
|------|------|
| 切换协议矩阵 / 切换协议 | 换法一览 / 切换步骤（在 CTA 里可用）/ 从 X 换成 Y 怎么做 |
| 讨论要点 / 与医生沟通 · 讨论要点 | 就诊前准备 / 与医生的讨论项 |
| 决策点 | 选择时机 / 分岔点 |
| 任务流 / 按时间顺序给出可执行步骤 | 从 X 到 Y（+ 自然段讲清楚顺序） |
| 剂量等效换算参考 | 剂量对照 / 不同途径的剂量对照 |
| 本指南面向 X | （删掉，正文直接开始） |

### H. 术语

- 医学中文（`绝对禁忌症`、`基线`、`谷值`）+ 品牌名（`补佳乐`、`色谱龙`）直接用，不加过度解释
- **社区黑话**（日雌 / 印雌 / 代购 / Dep）只在"在哪买"上下文使用，正文术语用正式名（`戊酸雌二醇注射液`）
- 英文缩写首次出现时括注全称：`VTE（静脉血栓栓塞）`、`AR（雄激素受体）`、`SHBG（性激素结合球蛋白）`，之后直用
- 剂量单位保持统一：mg / µg / pg/mL / ng/dL / mmol/L

## 5. 提交前 Checklist

写任何新 zh 页面或大规模改写前按顺序核查：

1. 读一遍 `before-you-start.mdx` 和 `dose-limits.mdx` 找语气
2. 章节标题：**全部名词短语吗？** 不是 → 重写
3. 开头段：**陈述句？不是对话？不是自我介绍？** 不是 → 重写
4. 列信息：**先找既有组件能不能承载**。能用就用组件，禁 `- ` 裸列
5. 全文 grep ASCII `"`、§ 4.G 禁用词表 → 全替换
6. 所有 `<DoseTable>` 列数 ≤ 4
7. 引用：每个医学主张有 `<CitationRef>` 吗
