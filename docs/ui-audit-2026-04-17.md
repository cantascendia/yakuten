## P0 — 立即修（违反 DESIGN.md 硬规则 / 可访问性硬伤）

### P0-1 · 缺少统一的主按钮（Primary CTA）样式
**DESIGN.md §5 (Buttons)** 要求：
> Primary: Clipped-corner shape. Background: `linear-gradient(135deg, #C84B7C, #E76395)`. Text: `on_primary` (Bold).

**现状**：全站没有 `.btn-primary` 类。Hero 上的「搜索 / 问 AI」入口是 `.hero-search`（glass panel + 边框），不是渐变主按钮；ActionCards 也只是玻璃卡片整块可点，没有传统 CTA。

**影响**：DESIGN.md 最关键的「Command Unit」视觉缺席 → 全站 CTA 语言层级缺失 → 用户对「这里就是操作入口」的感知弱。

**建议修复草案**（在 `src/styles/global.css` 新增）：
```css
.btn-primary {
  display: inline-flex; align-items: center; gap: var(--space-xs);
  padding: var(--space-sm) var(--space-lg);
  background: linear-gradient(135deg, var(--color-primary), var(--color-primary-container));
  color: var(--color-text-on-dark);
  font-family: var(--font-ui-accent);   /* Space Grotesk */
  font-weight: 600; letter-spacing: 0.05em; text-transform: uppercase;
  font-size: 0.875rem;
  clip-path: var(--clip-corner-sm);
  border: none; cursor: pointer;
  transition: filter .2s ease, transform .15s ease, box-shadow .2s ease;
}
.btn-primary:hover   { filter: brightness(1.08); box-shadow: 0 0 30px var(--color-primary-alpha-25); }
.btn-primary:focus-visible {
  outline: 2px solid var(--color-accent);   /* 金色 focus ring */
  outline-offset: 3px;
}

.btn-ghost {   /* DESIGN.md §5 secondary = ghost gold */
  display: inline-flex; align-items: center; gap: var(--space-xs);
  padding: var(--space-sm) var(--space-lg);
  background: transparent;
  color: var(--color-accent);
  border: 1px solid rgba(212, 168, 83, 0.5);   /* gold ghost border */
  font-family: var(--font-ui-accent);
  font-weight: 600; letter-spacing: 0.05em; text-transform: uppercase;
  clip-path: var(--clip-corner-sm);
  cursor: pointer;
  transition: background .2s ease, border-color .2s ease, box-shadow .2s ease;
}
.btn-ghost:hover   { background: rgba(212, 168, 83, 0.08); border-color: var(--color-accent); }
.btn-ghost:focus-visible { outline: 2px solid var(--color-accent); outline-offset: 3px; }
```

落地点（第一批）：Hero 的「问问 AI」副入口、DrugQuickNav 末尾的「浏览全部药品」、紧急事件页的「查看急救指南」—— 把原本裸链接升级成 `.btn-primary` / `.btn-ghost`。

### P0-2 · 焦点态可访问性缺失（Focus Visibility）
**WCAG 2.1 AA 2.4.7** 要求：所有可聚焦交互元素必须有对比度 ≥ 3:1 的可见焦点指示器。

**现状问题点**：
| 组件 | 文件 | 问题 |
|------|------|------|
| ReferenceLibrary tabs | `ReferenceLibrary.tsx:283` | 内联 `box-shadow: none !important; outline: none !important;` — **完全去除焦点环** |
| EmergencyBanner CTA | `EmergencyBanner.astro:92` | 无 `:focus-visible` 规则，依赖浏览器默认（但整块 a 标签常被 CSS reset 吞掉）|
| BloodTestChecker tabs | `BloodTestChecker.tsx:.btc-tab` | 仅改颜色，无 outline/ring |
| ReferenceLibrary filter tabs（非选中态） | 同上 | 无 focus-visible |
| 所有 Filter chips（DrugComparator presets / RiskScreener 选项） | 多个 | 依赖浏览器默认，被部分 reset 覆盖 |

**建议修复**：在 `global.css` 加一条兜底规则 + 逐个清理 `outline: none !important`：
```css
:where(button, a, [role="tab"], [role="button"]):focus-visible {
  outline: 2px solid var(--color-accent);
  outline-offset: 2px;
  border-radius: 0;   /* 保持非圆角 */
}
```
然后删掉 `ReferenceLibrary.tsx:283`、`BloodTestChecker.tsx:420` 里 `outline: none !important` 的 `!important`。

### P0-3 · 触点尺寸违反 WCAG 2.5.5（Touch Target ≥ 44×44）
| 组件 | 实际高度 | 文件 |
|------|----------|------|
| ReferenceLibrary tab | ~26px (`padding: 4px var(--space-sm); font-size: 0.75rem`) | `ReferenceLibrary.tsx:165` |
| DrugBrandIndex badge（若是可点） | ~18px | `DrugBrandIndex.tsx:320` |
| BloodTestChecker 「清除」按钮 | ~12px | `BloodTestChecker.tsx:340` |
| BloodTestChecker inline input | ~26px（输入框也是 touch 点） | `BloodTestChecker.tsx:247` |

**建议修复**：
- Tab / 小按钮统一最小高度：`min-height: 44px` + 增大 padding（或 `padding: var(--space-sm) var(--space-md)`，默认 8/16 → ~40px，近似 44）
- 纯视觉 badge 若非 clickable 则不适用，需先检查是否 `onClick`
- 输入框 `padding: 10px 2px` 起

### P0-4 · 切角 clip-path 不统一（DESIGN.md §4 Geometric Signature）
**DESIGN.md** 规定唯一签名切角：
```
polygon(0 0, 95% 0, 100% 5%, 100% 100%, 5% 100%, 0 95%)
```

**现状**：全站用了 5 套不同的 polygon：
| 值 | 定义位置 | 使用范围 |
|----|----------|----------|
| `0 0, 95% 0, 100% 5%, 100% 100%, 5% 100%, 0 95%` | `--clip-corner` | glass-card 基础类 |
| `0 0, 92% 0, 100% 12%, 100% 100%, 8% 100%, 0 88%` | `--clip-corner-sm` | EmergencyBanner CTA + 局部 |
| `0 0, 85% 0, 100% 15%, 100% 100%, 15% 100%, 0 85%` | `--clip-corner-accent` | 强调卡片 |
| `0 0, 92% 0, 100% 30%, 100% 100%, 8% 100%, 0 70%` | `InjectionCalculator.tsx` inline | 单点 |
| `calc(100% - 10px) 0, 100% 10px, …` | `BloodTestRange.astro` inline | 单点 |

**影响**：视觉节奏不齐；Stitch 原案主卡片用的是 `92%/12%`，跟我们 `--clip-corner`（`95%/5%`）不同，造成 Hero 卡片切角偏弱。

**建议修复**：
1. 对齐 Stitch：把 `--clip-corner` 改为 `polygon(0 0, 92% 0, 100% 12%, 100% 100%, 8% 100%, 0 88%)`（即合并现在的 `--clip-corner-sm`）。
2. 删除 `--clip-corner-sm`，全部合并到一个主切角。
3. `--clip-corner-accent` 保留作为强调变体（大标题卡）。
4. `InjectionCalculator.tsx` 和 `BloodTestRange.astro` 的 inline polygon 改成引用 `var(--clip-corner)` 或 `var(--clip-corner-accent)`。

这条是纯 CSS 改动 + 两处 inline 替换，风险极低。

## P1 — 建议修（影响体验但非阻断）

### P1-1 · Emergency CTA 应考虑贴合 DESIGN.md secondary 胶囊样式
现状 `.emergency-banner__cta` 用 `rgba(255,255,255,0.12)` + 白色边框。DESIGN.md §5 secondary = 金色 ghost。但红底上金色对比度不足（1.8:1）→ 现状白色 ghost 其实是合理的 AA 级让步。**保留现状，但补 focus-visible**：
```css
.emergency-banner__cta:focus-visible { outline: 2px solid #fff; outline-offset: 2px; }
```

### P1-2 · Hero 缺少 Stitch 的 pre-headline 小标签
Stitch 原案 Hero 主标题上方有一行：
```html
<div class="inline-block px-4 py-1 glass-panel text-secondary font-label tracking-[0.3em] text-sm">
  THE CELESTIAL PHARMACOPEIA
</div>
```

**现状**：`src/components/layout/HeroSection.astro:32` 的 `.hero-section__subtitle` 存在但视觉弱：
```astro
<span class="hero-section__subtitle glass-panel">{subtitle}</span>
```

**建议**：
- 内容上：`subtitle` 改为英文标签 "THE CELESTIAL PHARMACOPEIA"（或「循证 · 减害 · 引导就医」上方再加一行英文 pharma 标签）
- 样式上：`font-family: var(--font-ui-accent)` (Space Grotesk) + `letter-spacing: 0.3em` + `text-transform: uppercase` + `color: var(--color-accent)`
- 当前 subtitle 如果是中文的 slogan，则改成**双层 subtitle 结构**：英文小标签（Stitch 风格）+ 中文 tagline（保留现有）

### P1-3 · Hero slogan 视觉级差不足
Stitch 原案：副标题「愿此行，抵达真实的自己」用 Noto Serif SC + 金色 #D4A853 + 2xl/3xl + tracking-widest。
当前 `hero-section__slogan` 需要核查是否金色 + Serif —— 若是灰色 body，建议对齐 Stitch。

### P1-4 · 缺失 Mission Statement 统计区块
Stitch 原案的 `<section>` "MISSION STATEMENT"：
> 在中国，超过 **84%** 的跨性别激素使用者没有任何医疗指导。

数字 84% 用 5xl/7xl + secondary 金色，左侧 2px 金色竖条。

**现状**：首页没有对应区块。**建议新增** `src/components/layout/MissionStat.astro`（与 DrugQuickNav 同层插入），直接对齐 Stitch 布局 —— 这是高情感价值的转化段，比单纯 Hero → Cards → Footer 结构更有力。

### P1-5 · Hero CTA 入口不够「产品级」
现状 Hero 只有一个搜索入口 + 一个 AI 入口（两个 glass panel）。Stitch 原案则把主导航下移到「三张 ActionCard」，直接代替传统 CTA。

**两个方案择一**：
- **A 方案**（推荐）：保留搜索 + AI 两个入口，但加一行显式 `.btn-primary` "开始评估" → 跳转 `/zh/pathway/`（已有页面），强化 CTA。
- **B 方案**：让 Hero 精简，把 ActionCards 升为 Hero 的一部分（hero-actions），滚动感更紧凑。

### P1-6 · Action Card 的 CTA 文案样式不一致
`stitch-exports` 的 ActionCard 底部有：
```html
<div class="flex items-center gap-2 text-[#4CAF50] font-label text-sm tracking-wider uppercase">
  Explore Baseline <arrow_forward>
</div>
```

**现状**：`src/components/layout/ActionCards.astro` 的 CTA row 需要核查 `font-family` 是否已用 `var(--font-ui-accent)` —— 审计数据显示是 line 236 已使用，但 `text-transform: uppercase` + `letter-spacing` 建议再加，让它看上去像 Stitch 的 `tracking-wider uppercase`。

## P2 — 长期打磨

### P2-1 · Particle 背景从 CSS 升级为 Canvas
DESIGN.md §5 要求「floating Light Particles at 5% opacity」—— 动态漂浮感。
现状 `src/styles/global.css:235` 是静态 radial-gradient，不会漂。
**建议**：实现 `<ParticleCanvas />` 岛组件（client:idle，max 60 粒子，respects `prefers-reduced-motion`）。性能可控；做成可选组件，首页 `<main>` 后追加即可。

### P2-2 · 删除少量硬编码 fallback 颜色
`DrugBrandIndex.tsx:320–323` 里 `#2d8a4e / #2563eb / #dc2626` 这三个 Starlight fallback hex 可以移到 `--color-*` 语义变量或直接用 `var(--color-safe/--color-primary/--color-danger)`。
`AIAssistant.tsx:154` 的 `#1a1625` 同理改成 `var(--color-surface-container-low)`。

### P2-3 · 输入框 focus 态改成渐变底线（DESIGN.md §5）
> Focus State: The bottom border transforms into a `primary` gradient.

现状 `BloodTestChecker.tsx:420` 的 focus 仅是 `border-bottom-color: var(--color-primary)`（实色）。
**建议**：用 `border-image-source: linear-gradient(135deg, var(--color-primary), var(--color-primary-container))` + `border-image-slice: 1`，或用伪元素 `::after` 画一条 2px 渐变底线叠加。
ROI 一般，放最后。

### P2-4 · Filter chip 选中态对比度
`ReferenceLibrary` 的 `tabActive` 用 `background: var(--color-primary-alpha-15)` + primary border —— 颜色对比是 OK 的，但在 glass panel 底色上有时候看起来偏灰。建议 active 态改用渐变轻 tint：`background: linear-gradient(135deg, var(--color-primary-alpha-15), var(--color-primary-alpha-25))` —— 视觉"按下感"更足。

### P2-5 · Loading / Skeleton 态视觉
所有 React 岛（DoseSimulator / DrugComparator / BloodTestChecker）首屏 client:visible 之前应有 skeleton 占位 —— 当前部分组件闪现空白。这属于前端工程动作，但与 UI 感知相关，列在这里。
